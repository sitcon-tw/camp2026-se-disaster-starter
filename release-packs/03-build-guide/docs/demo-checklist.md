# Demo 檢查表

請在完成 v1 後，用這份檢查表確認成果能被展示。

## 1. 首頁入口

- [ ] Demo 可以從首頁操作。
- [ ] 主要畫面有接進 `src/app/App.tsx` 或由它匯入的 component。
- [ ] 沒有只存在於未被引用 component 的成果。

## 2. 對應使用者取捨

- [ ] Demo 看得出我優先服務的使用者。
- [ ] Demo 有對應 `docs/decisions.md` 中的主要取捨。
- [ ] Demo 沒有試圖同時服務所有人。

## 3. 對應流程圖

- [ ] Demo 的主流程可以對應 `docs/flow.md`。
- [ ] Demo 沒有跳過流程圖中的人工確認點。
- [ ] Demo 沒有把不能自動處理的分支直接變成成功結果。

## 4. 不確定性呈現

- [ ] 畫面有清楚標示至少一個需要人工確認的地方。
- [ ] 畫面有清楚標示至少一個不能直接相信的資訊。
- [ ] 畫面沒有把未確認資訊顯示成已確認。
- [ ] 畫面沒有把資訊取得方式當成查核結果。

## 5. 操作或判斷紀錄

- [ ] Demo 中有至少一筆操作、判斷或處理紀錄。
- [ ] 紀錄能說明為什麼這樣判斷。
- [ ] 如果資訊被拒絕或暫時不採用，有留下理由。

## 6. AI 使用與人類判斷

- [ ] `docs/ai-log.md` 有記錄 Codex 做了什麼。
- [ ] 有記錄我採用或不採用什麼 AI 建議。
- [ ] 沒有讓 AI 自動決定資訊真偽、救災優先順序或派工。

## 7. 基本驗證

請至少完成一項：

- [ ] 本機 demo 跑得起來。
- [ ] GitHub Pages demo 跑得起來。
- [ ] `pnpm build` 通過。
- [ ] `pnpm test` 通過。
- [ ] 我知道目前哪個檢查還沒通過，並已寫在 `docs/ai-log.md` 或 `docs/decisions.md`。

## 展示時請準備說明

請用 30 秒說明：

```text
我的 v1 主要服務＿＿＿＿。
我最想避免的錯誤是＿＿＿＿。
我用流程圖讓 Codex 幫我實作＿＿＿＿。
目前仍不確定或未支援的是＿＿＿＿。
```
