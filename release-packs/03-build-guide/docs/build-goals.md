# v1 實作目標

> 這份文件不是功能清單，而是 v1 prototype 的方向盤。請根據自己的 `docs/decisions.md` 和 `docs/flow.md` 取捨實作。

## v1 的一句話目標

請先填：

```text
我的 v1 要幫＿＿＿＿，在看到原始資訊時，可以＿＿＿＿，並避免＿＿＿＿。
```

範例：

```text
我的 v1 要幫資訊整理者，在看到原始資訊時，可以判斷哪些資訊需要人工確認，並避免把模糊資訊直接變成可行動任務。
```

## v1 最低要做到

請讓你的 prototype 至少做到以下五件事：

1. 使用者能看到原始資訊。
2. 使用者能看出哪些資訊仍不確定或需要人工確認。
3. 使用者能看出哪些資訊不能直接相信或不能直接變成任務。
4. 使用者能根據你的流程做出一個候選結果、判斷結果或整理結果。
5. 使用者能看到至少一筆操作、判斷或處理紀錄。

## v1 不必做到

今天不要求：

- 不要求處理所有資料。
- 不要求完整任務系統。
- 不要求登入。
- 不要求後端。
- 不要求資料庫。
- 不要求真實地圖。
- 不要求真實通知。
- 不要求漂亮 UI。
- 不要求讓 AI 自動判定真偽或救災優先順序。

## 請依照自己的取捨選擇重點

請勾選你 v1 最重視的 1–2 項：

- [ ] 讓回報者更容易送出不完整但有價值的資訊。
- [ ] 讓資訊整理者看出哪些資訊需要人工確認。
- [ ] 讓行動者不要誤信未確認資訊。
- [ ] 讓原始資訊與整理後判斷可以對照。
- [ ] 讓操作或判斷紀錄更清楚。
