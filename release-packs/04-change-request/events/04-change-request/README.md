# 新事件：現場資訊改變了

這個資料夾包含課中釋出的新事件。

請注意：

- 這些資料是外部新傳入的未整理資料。
- 它們不一定符合你目前設計的資料格式。
- 它們不一定可信。
- 它們不應該直接被顯示成已確認事實。
- 它們的目的不是給你標準答案，而是測試你的 v1 prototype 面對變更時是否還能說得清楚。

## 你要做什麼

1. 先閱讀 `incoming-data.json`。
2. 選擇至少一個事件處理。
3. 用 `docs/prompts/change-impact-analysis.md` 請 Codex 做影響分析。
4. 更新 `docs/change-notes.md`。
5. 決定要支援、部分支援，還是暫時不支援。
6. 若要修改前端，請只做最小改版。

## 建議處理策略

你可以選擇：

- 支援：讓 prototype 顯示新事件的處理結果。
- 部分支援：只處理一部分，其餘標示為需要人工確認。
- 暫時不支援：不改 code，但在文件中明確記錄原因。

不管選哪一種，都要說清楚為什麼。
