# 新事件任務

請從 `incoming-data.json` 中選擇至少一個事件處理。

若你有足夠時間，也可以處理兩個事件，但不要為了處理更多事件而失去判斷品質。

## 任務步驟

1. 選擇一個事件。
2. 閱讀該事件的 `humanConcern` 與 `questionsForTeam`。
3. 請 Codex 根據 `docs/prompts/change-impact-analysis.md` 做影響分析。
4. 更新 `docs/change-notes.md`。
5. 決定：支援、部分支援，或暫時不支援。
6. 如果要修改前端，只做最小改版。
7. 更新 `docs/ai-log.md`。

## 最低完成條件

請至少完成：

- [ ] 選定一個事件 ID。
- [ ] 寫出它打破了你原本哪個假設。
- [ ] 寫出哪個地方需要人工確認。
- [ ] 寫出你決定支援、部分支援或暫時不支援。
- [ ] 寫出至少一個不採用的 AI 建議，或一個你自己補充的判斷。

## 如果你修改前端

請確認：

- [ ] 變更結果能從首頁看到。
- [ ] 不確定資訊沒有被顯示成已確認。
- [ ] 沒有直接把 `events/**` 的資料搬進整理後資料。
- [ ] `docs/ai-log.md` 有記錄 Codex 修改了什麼。

## 如果你不修改前端

也可以接受，但請確認：

- [ ] `docs/change-notes.md` 有清楚說明為什麼不修改。
- [ ] `docs/decisions.md` 或 `docs/change-notes.md` 有記錄目前限制。
- [ ] 你能向小隊說明這個取捨。
