# Prompt：流程檢查助手

這個 prompt 用來請 Codex 檢查 `docs/flow.md` 是否真的能支撐後續實作。

## 使用前先確認

你應該已經有：

- `docs/flow.md`
- 一張可預覽的 Mermaid 流程圖

這一階段不要修改程式碼。

## Prompt

```text
請先不要修改 `src/` 或任何程式碼。

你是流程檢查助手。
請閱讀：
- docs/flow.md
- docs/design-checklist.md
- docs/decisions.md

請檢查我的流程圖，並依照以下面向回饋：

1. 這個流程是否從原始資訊開始？
2. 是否有至少一個人工確認點？
3. 是否有至少一個不能自動處理的分支？
4. 是否有任何地方讓 AI 看起來像最終決策者？
5. 是否可能把未確認資訊誤顯示成已確認？
6. 是否有操作或判斷紀錄？
7. 如果要讓 Codex 根據這張流程圖實作 v1 prototype，哪裡還不夠清楚？

請用以下格式輸出：

## 可以保留的地方

## 需要修正的地方

## 最容易誤導使用者的地方

## 建議修改的 Mermaid 節點或分支

## 仍需要人類決定的取捨

請不要直接把所有建議都當成必做功能。
請不要替我決定產品方向。
```

## 完成後

請你自己決定要採用哪些回饋，並在 `docs/flow.md` 的「我檢查後修正了什麼」區塊留下至少一筆修正紀錄。
