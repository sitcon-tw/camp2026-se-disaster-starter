# 流程設計檢查表

請用這份檢查表檢查 `docs/flow.md`。

你不需要每一題都寫很多字，但每一題都要看過一次。

## 1. 使用者與目標

- [ ] 流程有對應 `docs/decisions.md` 中選定的主要使用者。
- [ ] 流程有幫這個使用者完成一件明確的事。
- [ ] 流程沒有試圖同時服務所有人。

## 2. 原始資訊與不確定性

- [ ] 流程從「原始資訊」開始，而不是從整理後資料開始。
- [ ] 流程有保留原文或原始資訊的概念。
- [ ] 流程有處理資訊不足、來源不明或內容模糊的情況。
- [ ] 流程沒有把未確認資訊直接當成已確認。

## 3. 人工確認

- [ ] 流程中至少有一個人工確認點。
- [ ] 流程有說明什麼情況需要人工確認。
- [ ] 流程沒有讓 AI 自動決定資訊是否為真。
- [ ] 流程沒有讓 AI 自動決定救災、派工或行動優先順序。

## 4. 不能自動處理的分支

- [ ] 流程中至少有一個「不能直接處理」或「暫時不採用」的分支。
- [ ] 流程有說明為什麼不能直接處理。
- [ ] 流程沒有把所有輸入都強迫變成候選結果。

## 5. 操作或判斷紀錄

- [ ] 流程中有留下操作或判斷紀錄。
- [ ] 紀錄能回答「誰做了什麼判斷」或「為什麼這樣判斷」。
- [ ] 如果資訊被拒絕或暫時不採用，流程仍保留理由。

## 6. 圖能不能幫助 Codex 實作

- [ ] 流程圖有清楚的開始點。
- [ ] 流程圖有清楚的結束點或輸出。
- [ ] 判斷分支的文字不含糊。
- [ ] Mermaid 可以在 VS Code 預覽成功。
- [ ] 如果讓 Codex 根據這張圖實作，它不需要再猜主要流程。

## 檢查後請補到 `docs/flow.md`

請至少補一項：

```text
我檢查後修正了什麼：
原本：
修正後：
為什麼：
```
