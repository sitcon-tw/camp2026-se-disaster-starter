# 第一階段學生脈絡

今天做的是前端-only 的學習原型，不是真實災害應變系統。

你現在不需要理解後續階段，也不需要知道完整課程安排。你只需要先面對一批原始資訊，嘗試做出能幫下一位協作者理解資訊的前端工作台。

第一階段的目標不是建立正式資料模型，也不是做完整產品。你正在嘗試做出一個會暴露資料品質問題的前端原型。

## 現在要記得的事

- 成功不是分類正確，而是把「為什麼現在還不能判斷」說清楚。
- 你會使用 coding agent，但 agent 的整理結果需要人類檢查。
- 不確定資訊不能被顯示成 confirmed / verified 事實。
- 原始資訊可以被嘗試整理，但不能假裝已經被確認。
- 如果資料缺少取得方式、時間、地點、當事人或確認狀態，工作台應該把問題顯示出來。

## 不做的事

- 不做後端。
- 不呼叫外部 API。
- 不使用 runtime LLM。
- 不查真實地圖、真實地址、真實電話或真實人物資料。
- 不做真實救災判斷。
